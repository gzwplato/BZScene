Description

Description et manipulation de quaternions

Les quaternions sont un système numérique qui étend les nombres complexes     et sont appliqués à la mécanique dans l'espace tridimensionnel.
    Une des caractéristiques des quaternions est que la multiplication de deux quaternions n'est pas commutative.

class operator *(constref A, B: TBZQuaternion): TBZQuaternion;

Retourne le produite de deux quaternion.
"quatrenion_Left * quaternion_Right" est la concaténation d'une rotation de A suivie de la rotation B
Note : L'ordre inversé est important A * B car B * A donne de mauvais résultats !

class operator =(constref A, B: TBZQuaternion): Boolean;

Retourne True si le quaternion A est égale au quaternion B

class operator <>(constref A, B: TBZQuaternion): Boolean;

Retourne True si le quaternion A n'est pas égale au quaternion B

function ToString: String;

Retourne une chaine de caractères formaté représentant le quaternion : "("x, y, z, w")"

function ConvertToMatrix: TBZMatrix;

Convertis le quaternion en une matrice de rotation (possibilité au quaternion d'être non-unitaire).
On suppose que la matrice sera utilisé pour multiplié la colonne du vecteur sur la gauche :
vnew = mat * vold.
Fonctionne correctement pour les coordonées "Main droite".

procedure Scale(ScaleVal: single);

Redimensionne le quaternion.
Si le facteur de mise à l'échelle Un facteur d'échelle est appliqué à un quaternion puis la rotation mettra à l'échelle les vecteurs lors de leur transformation.
On suppose que le quaternion est déja normalisé.

function MultiplyAsSecond(const qFirst : TBZQuaternion): TBZQuaternion;

Retourne le produit de deux quaternion : quaternion_Right * quaternion_Left.
Pour combiner les rotations, utilisez Muliply(qSecond, qFirst), ce qui aura pour effet de faire une rotation par qFirst puis qSecond.
Note : l'ordre de la multiplication est importante !

function Slerp(const QEnd: TBZQuaternion; Spin: Integer; t: Single): TBZQuaternion; overload;

Interpolation linéaire sphérique de quaternions unitaires avec des tours (spins).
Note :
- la valeur de début QStart correspond au quaternion courrant Self.
- t est une valeur comprise entre 0.0 et 1.0
- Spin est le nombre de rotation supplémentaire à effectuer.
