BZScene bibliothèque multimédia 2D, 3D et audio pour Larazus & FPC

Licence : MPL / GPL

Preface

BZScene est la solution Audio, 2D et 3D pour développer des applications graphiques multiplateformes avec Lazarus &amp; FPC 32 et 64 bits.

BZScene est en partie basé sur le moteur de rendu GLScene. Communauté dont je fais partie. GLScene est le fruits de centaines de développeurs à travers le monde.

Son histoire débute dans les années 1995 avec Mike Lischke( "Soft Gems - Mike Lischke") et ces bibliothèques pour delphi ici et là entre autres. Tout cela afin de disposer d'une gestion des bitmaps simple, rapide et robuste avec Lazarus. Mais aussi d'améliorer le portage de GLScene depuis Delphi.

Que contient BZScene ?

BZScene est divisé en neuf groupes principaux, afin de faciliter le débogage et le développement en général.

  • Core : Contient les unités communes à la bibliothèque

  • Dependencies : Contient les unités externes utilisée, comme par exemple Freetype, PasJpeg...

  • Maths : Contient les unités découlant de mathématique vectorielles et de base. Ces unités sont optimisée et utilisent les instructions SIMD : SSE/AVX

  • Image : Contient les unités relatives à la gestion et la manipulation des bitmaps

  • Audio : Contient les unités relatives à la gestion du son

  • Utils : Contient des unités qui regroupent des méthodes utiles dans différents domaines

  • Engine : Contient les unités des moteurs de rendu 2D et 3D. "encore en cours de dev"

  • Components : Contient les unités des composants pouvant être installés dans l'EDI

  • DevTools : Contient des unités utiles pour le développeur



Les principales unités à connaitre. Ce quelles contiennent et leurs capacités

  • Options de compilation Afin de contrôler les options de compilation vous devrez éditer le fichier "BZScene_options.inc"
    Vous pourrez contrôler l'utilisation des méthodes rapide mathématique ainsi que l'utilisation de l'assembleur

  • Core :

    1. BZSytem.pas :
      L'unité BZSystem, regroupe quelques routines pour la detection de l'OS, du CPU et autres informations sur le systeme en général. Elle contient également des fonctions utiles pour une application. (retrouver le dossier de l'application, dossier temporaire de l'OS etc...)

    2. BZUtils.pas :
      L'unité BZUtils contiens des routines utiles dans différents domaines tel que les nombres, la manipulation de bits, le calcul de CRC32 etc...
      Certaines routines ont étés optimisées gràce à l'utilisation de l'assembleur.

    3. BZClasses.pas :
      L'unité BZClasses contient des classes conçues à des fins génériques.
      Elles implémentent un mécanisme de persistance légèrement différent de celui de la LCL / VCL.
      Elles permettent le contrôle de version au niveau de l'objet avec une persistance 100% ascendante et une compatibilité polymorphe.
      Cette unité, contient également les définitions d'objets à remplacer.
      Ces objets peuvent être 'notifiés pendant les changements' et peuvent être animées (simulations basées sur le temps).
      Elles peuvent inclure un méchanisme de lecture/écriture de données dans un flux optimisée.
      Elle contient également des classes spécialisées et utiles comme par exemple,le remplacement d'un TStringList basé sur le hachage.

      Les principales classes à connaitre :

      • TBZPersistentObject : Classe de base pour les objets persistants.
        L'exigence de base est l'implémentation de ReadFromFiler et WriteToFiler dans les sous-classes.
        L'autre exigence est l'utilisation d'un constructeur virtuel, ce qui permet la construction polymorphe (ne pas oublier d'enregistrer vos sous-classes).
        Les avantages immédiats sont le support du streaming (flux, fichier ou chaîne), l'affectation et le clonage.

      • TBZPersistentObjectList : Liste d'objets persistants.
        Similaire à TList, mais fonctionne sur des objets TObject directement et dispose d'un mécanisme de persistance des données.
        Contrairement à la TObjectList de la VCL / LCL, TBZPersistentObjectList ne libère pas ses objets lors de la destruction ou de Clear. Vous devez utiliser les fonctions Clean et CleanFree.

        • Seuls les éléments TBZPersistentObject et descendants seront gérés correctement.

        • La liste peut être utilisée sous forme de LIFO avec Push and Pop.

        • Vous pouvez effectuer des opérations booléennes de base.

        Note : L'implémentation de la fonction IndexOf est jusqu'à 3 fois plus rapide que celle de TList

      • TBZUpdateAbleObject : Classe abstraite décrivant l'interface "IGLZNotifyAble"
        Elle offre un systeme d'événements de notifications, qui donne la capacité de notifer les changements de valeurs des propriétés à d'autres classes.

      • TBZCadenceAbleComponent : Classe abstraite décrivant l'interface "IBZProgessAble"
        Elle est basée sur le Temps, permettant de créer des animations / simulations (2D,3D,...) ou autres fonctions d'arrière-plan

      • TBZUpdateAbleComponent : Classe abstraite décrivant et supportant IGLZProgessAble et IGLZNotifyAble
        Elle offre la cappacité de notifer les changements de valeurs des propriétés à d'autres classes. Et permet "d'animer" les propriétés en se basant sur le temps

      • TBZCustomDataFile : Classe de base abstraite pour les interfaces de formats de fichiers de données.
        Cette classe déclare les comportements de base liés aux fichiers. C'est à dire, pouvoir charger / enregistrer à partir d'un fichier ou d'un flux.
        Il est fortement recommandé de surcharger SEULEMENT les méthodes d'acces à la mémoire. (fonction CanLoad et procedure LoadFromMemory)
        Les comportements en flux permettent d'ajouter d'autres capacités en Entrée/Sortie, comme la compression, la mise en cache, etc... sans qu'il soit nécessaire de réécrire de sous-classes.
        Note : CheckFormat / LoadFromMemory / SaveToMemory : Les enfants doivent réécrirent ces méthodes pour supporter la lecture et ou l'écriture d'un format de fichier spécifique.

      • TBZStringList : Classe optimisée pour utiliser une liste de chaine de caractères. A utiliser en remplacement d'une TStringList.
        On a la possibilité d'associer un d'objets et ou un Tag de type integer et single à chaque élément. A chaque item est associé à un "Hash" permettant ainsi de retrouver un élément spécifique très rapidement juste par correspondance. La liste n'a donc pas besoin d'être triée pou effectuer un recherche rapide
        Cette classe dispose également d'une méthode de trie 'DirectSort" plus rapide que QuickSort. Elle utilise l'algorithme de tri "A-Sort".

    4. BZArrayClasses.pas :
      L'unité BZArrayClasses contient des classes génériques pour manipuler des tableaux "Array" a travers un pointeur.
      Elle regroupe des tableuax 1D et 2D pour les types Byte, Integer, Single et double
      Fournis quelques fonctions utiles et communes ajouter, supprimer, l'accès aux données par incrément, de tri, l'accès en mode LIFO (Stack), la "mutation"

      Les principales classes et types à connaitre :

      • Tableaux prédéfinis 1D : TBZByteList, TBZIntegerList, TBZSingleList, TBZDoubleList

      • Tableaux prédéfinis 2D : TBZByte2DMap, TBZInteger2DMap, TBZSingle2DMap, TBZDouble2DMap

    5. BZColors.pas :
      L'unité BZColors définis et regroupe les systèmes de couleurs RGBA / BGRA suivant l'OS, HSV, et autres espaces de couleur et leurs informations.
      Contient des objets pour la gestion de palette de couleurs et des dégradés.

      Les principales classes et types à connaitre :

      • TBZColor32 / TBZColor TBZColor32 est un enristrement avancé décrivant le format de couleur RGBA ou BGRA automatiquement en fonction de l'OS. Les valeurs sont de type byte [0,255]. TBZColor est un type de convenance décrivant un TBZColor32
        TBZColor32 supporte les opérateurs courrant : +, -, *, /, and, or, xor. Il contient également plusieurs méthode pour facilité la manipulation de couleur. L'acces se fait via les propriétés Red, Green, Blue, Alpha ; AsInteger, et un tableau de valeur et enfin AsVector3b (x,y,z).

      • TBZColorVector Identique à TBZColor32 sauf que les valeurs sont de type single et sont comprises dans l'interval [0.0,1.0].

      • TBZPaletteEntries Définis un tableau de TBZColor32, pour stocker rapidement une palette de couleur.

      • TBZColorList TBZColorList est une classe servant à gérer une liste de couleur.

      • TBZGradientColorList TBZGradientColorList est une liste servant à décrire un dégradé de couleur.

      • TBZColorsManager TBZColorsManager est une classe permettant de gérer un liste de couleur et les dégradés de couleur. Elle permet l'animation de la palette.

    6. BZGraphic.pas :
      L'unité BZGraphic définis et regroupe des enregistrements et objets de base pour la gestion des bitmaps, des informations sur les formats d'image,
      Définis les types de filtres et autres données utiles à la gestion du graphisme.
      Elle contient également quelques méthodes globale utiles.

      • TBZRect / TBZFloatRect Ces deux types sont des enregistrements avancés permet la manipulation de "Rectangle" comme le TRect natif.
        Ces types supportent les operateurs, ainsi que des méthodes utiles à leur manipulation.

      • TBZRawImageData / TBZRawImageDescription Ces deux classes sont utilisées par TBZCustomBitmap, elles permettent de décrire le format et les spécification d'image chargée via "LoadFromFile".

      • TBZCustomBitmap TBZCustomBitmap est la classe de base pour la gestion de bitmap. Elle contient des méthodes basiques pour copier en entier ou en partie un autre bitmap. Dessiner des lignes, des rectangles, déplacer ("Scroller") le bitmap, et de quelques méthodes de traitement. Il dispose également d'un systeme de gestion de calques.

      • TBZImageFileFormatsList TBZImageFileFormatsList permet de stocker les différentes classes en charge de la lecture et de l'écriture d'image dans différents formats. Sont accès se fait globalement vi la fonction GetBZImageFileFormats .

    7. BZParallelThread.pas :
      L'unité BZParallelThread contient un objet TBZParallelThread et des méthodes globales qui permettent d'exécuter des opérations en parallele facilement grâce au multithreading. Elle dispose de la possibilité de suivre la progression de la tâche.

  • Maths :

    1. BZMath.pas L'unité BZMath contient des fonctions mathématiques optimisées.
      Si l'option USE_FASTMATH est activée, des fonctions optimisées par approximation sont utilisées.

      Par exemple:
         - Les fonctions 'approximatives' de Sin et Cos ont une précision moyenne de 4E-8 et sont jusqu'a 5 fois plus rapide que les fonctions natives.

        Elle comprend également des fonctions trigonométriques, calcul d'interpolations,    Des fonctionnalités avancées comme Bessel, BlackMan. Et aussi quelques fonctions utiles   Il y a également des fonctions indéfinies dans l'unité Math FPC tel   ArcCsc, ArcSec, ArcCot, CscH, SecH ect ...

    2. BZVectorMath.pas :
      L'unité BZVectorMath contient des enregistrements avancés optimisée pour l'utilisation de mathématique vectorielle. Une majorité des méthodes utilisent l'accélération SIMD (SSE, SSE3, SS4, AVX, AVX2) si l'option USE_ASM_OPTIMIZATIONS est activée.
      Elles peuvent être utilisées dans les graphiques 2D / 3D et tout autre application nécessitant des calculs mathématique avec des vecteurs.

      Les types de vecteurs supportés :

      • Vecteur 2D Integer, Single et Double

      • Vecteur 3D Byte, Integer, et Single

      • Vecteur 4D Byte, Integer, Single @TODO Double (AVX & native only)

      • Matrice 2D Single

      • Matrice 4D Single Matrix

      • Quaternion

      • Plan Homogène

      Les liste de vecteurs supportés :

      • TBZVector2f2DMap

      • TBZVector4f2DMap

      • TBZVector2iList

      • TBZVector2fList (supporte le trie en fonction des X puis Y)

      • TBZArrayOfFloatPoints

      • TBZArrayOfPoints

      Contient également des fonctions identique au script GLSL / HLSL tel que Lerp, SmoothStep, Step...

    3. BZVectorMathEx.pas :
      L'unité BZVectorMathEx contient des enregistrements avancés supplémentaire à BZVectorMath optimisée avec les instructions SIMD utiles principalement en 3D.

      Types supportés :

      • Matrix affine (pour la manipulation en 2D)

      • Frustum

      • Bounding Box 3D orienté (OBB)

      • Bounding Sphere

      • Axis Aligned Bounding Box for 3D, Axis Aligned Bounding Rect for 2D

      • Raycast

    4. BZVectorMathUtils.pas :
      L'unité BZVectorMathUtils contient des méthodes pour faciliter l'utilisation des vecteurs

  • Image :

    1. BZBitmap.pas :
      L'unité BZBitmap, contient la classe TBZBitmap descendant de TBZCustomBitmap. Elle est spécialisée dans la gestion et la manupilation et le traitement d'image.
      C'est une sorte de **Super TBitmap**.
      La manipulation se fait exclusivement en 32 bits (RGBA/BGRA)

      Capacités :

      • Accès aux pixels optimisé

      • Accéder aux données graphique indépendemment des routines de FPC et de la LCL

      • Affichage 32bit avec ou sans transparence en fonction du sytsteme et de l'interface.

      • Importation / exportation des divers formats d'image (cf l'unité BZBitmapIO)

      • Support des masques.

      • Transformations (Mirroir Horizontal et vertical, rotation, recadrage, redimensionnement (Normal, lissé, lissage bicubique, par rééchantillonage avec + de 30 filtres disponibles))

      • Fusion d'image +20 modes différents

      • Conversion en niveaux de gris suivant plusieurs méthodes

      • Filtres sur les couleurs + de 50 (negate, modulate, luminosité, saturation...)

      • Lissage (rapide, lineaire, box, split, gaussien, radial, motion, FXAA)

      • Filtres de convolutions + de 60 filtres prédéfinis disponibles

      • Filtres de deformations. (Tourbillon, pincement, fisheye, polaire/cartésien, sinuosidale, diffusion)

      • Filtres de seuillage.

      • Filtres de morphologique.

      • Filtres de segmentation, detection des contours ( par gradient, canny, DOG, LOG,...)

      • Filtres d'effets spéciaux. (Bloom, Gloss, Pixelate...)

      • Dessiner des points, lignes, cercles, ellipses, rectangle, triangle, beziez remplissage (uni, dégradé, texture)

      • Afficher du texte (via la bibliothèque FreeType)

      • Génération de texture (Damier, grille, bruit)

    2. BZBitmapIO.pas :
      L'unité BZBitmapIO est une unité de convenance pour la prise en charge de tous les formats image supportés

      Format supportés en lecture :

      • BMP Windows et OS/x. 1,2,4,8,16,24,32 et 64 bits. bmp, dib

      • X PixMap. xpm

      • Portable pxmap : pbm, pgm, pnm, ppm, pam, pfm

      • PCX ZSoft Paintbrush. 1, 2, 4, 8, 24, 32 bits. pcx, pcc, scr

      • TGA TrueVision TARGA. 8, 16, 24, 32 bits. tga, vst, icb, vda

      • Graphic Interchange Format 87a et 89a, animé ou non. gif

      • Potable Network Graphic. 1 à 64 bits. png

      • JPEG. jpeg, jpe, jpg, jfif

      • Google WEBP, webp


      Format supportés en écriture à l'heure actuelle :

      • BMP Windows 24 & 32 bits uniquement

      • TGA

      • JPEG

    3. BZImageViewer.pas :
      L'unité BZImageViewer contient le composant TBZImageViewer qui est un control visuel similaire au TImage
      Il dispose de quelques options supplémentaires, tel que la prise en charge du déplacement, du zoom. L'affichage d'un fond de type echiquier.
      Supporte plusieur filtre pour la mise à l'échelle et différents comportements en fonction de la taille de l'image chargée.

  • Utils :

    1. BZRandomGenerator.pas :
      L'unité BZRandomGenerator contient un classe "Thread-Safe" pour la génération de nombre aléatoire.

    2. BZKeyboard.pas :
      L'unité BZKeyboard contient des fonction utiles pour l'interception et le mappage des touches appuyées sur le clavier.

    3. BZInterpolationFilters.pas :
      L'unité BZInterpolationFilters contient plusieurs type de filtre d'interpolation (utilisé pour le rééchantillonage d'image par exemple)

    4. BZAnimationTool.pas :
      L'unité BZAnimationTool contient un objet et des méthodes pour le calcul d'animations de valeurs gràce à des fonctions paramétrique 1D (easing)

    5. BZGeoTools.pas :
      L'unité BZGeoTools contient des classes utiles pour la manipulation de primitives vectorielles.

      • Lignes

      • Polyline

      • Polygones

      • Cercle/Ellipse

      • Courbes de bezier quadratic, cubic, spline

    6. BZNoiseGenerator.pas :
      L'unité BZNoiseGenerator contient des classes d'objet permettant de générer divers bruits :

      • Bruit blanc, bruit gaussien

      • Bruit de Perlin

      • Bruit par valeurs

      • Bruit Simplex / OpenSimplex


      NOTE : Celle-ci est finaliser pour une utilisation à 100% et pour générer des textures de bruit cohérent

  • Audio :

    1. BZSound.pas :
      L'unité BZSound contient les classes et collections pour la gestion des sources audio et des effets sonore.

    2. BZOpenALManager.pas :
      L'unité BZOpenALManager contient le gestionnaire audio pour le moteur OpenAL.

  • Components :

    1. BZThreadTimer.pas :
      L'unité BZThreadTimer contient un composant non-visuel TBZThreadTimer. C'est un bon remplaçant au TTimer habituel. Celui est plus précis et s'éxécute en arrière-plan dans un thread à part.

    2. BZHotKeyManager.pas :
      L'unité BZHotKeyManager contient trois composants non-visuel TBZFormHotKey, TBZApplicationHotKey, TBZSystemHotKey ceux-ci permettent l'interception de l'appuis de raccourcis clavier chacun à leur niveau. Dans la fenêtre en cours, dans toutes l'application, en dehors de l'application (Application minimiser dans la barre d'icone par exemple)

    3. BZCadencer.pas :
      L'unité BZCadencer contient un composant non-visuel TBZCadencer à la façon d'un TTimer, en plus précis et spécialisé dans la mise en place d'animations synchronisées.

    4. BZFileFinder.pas :
      L'unité BZFileFinder contient un composant non-visuel TBZFileFinder d'aide à la recherche de fichiers récursif, sur un disque ou dans un dossier.

    5. BZScreenMode.pas :
      L'unité BZScreenMode contient un composant non-visuel TBZScreenMode permmettant de changer la résolution du moniteur et d'afficher en mode "plein écran"

  • DevTools :

    1. BZTypesHelpers.pas :
      L'unité BZTypesHelpers surcharge les assistants de types (byte, word, integer, string, TDatetime) par défaut de FPC et Lazarus en y ajoutant plusieurs fonctions utiles.

    2. BZLogger.pas :
      L'unité BZLogger contient un composant non-visuel TBZLogger permmettant de générer des "log" très simplement dans un fichier, dans la console, dans un fenêtre supplémentaire etc...

    3. BZStopWatch.pas :
      L'unité BZStopWatch contient un composant non-visuel TBZStopWatch est une sorte de chronomètre très précis, qui permet de calculer le temps en 2 actions.

  • Engine :

    1. BZCustomShader.pas :
      L'unité BZCustomShader contient les classes de base à hériter pour le rendu de shader (openGL/Vulkan/Software)

    2. BZBitmapRasterizer.pas :
      L'unité BZBitmapRasterizer contient des classes spécialisées dans le rendu de shader alla GLSL en software.

    3. BZBitmapFont.pas :
      L'unité BZBitmapFont contient les classes pour le rendu de police bitmap.

    4. BZSoftwareShader_?????.pas :
      Ces unités contiennent les methodes de rendu de différents Shader en software.

Prérequis

La principale dépendance pour utiliser les bitmaps est la bibliothèque FreeType.
Que ce soit ous Linux ou Windows. Cette bibliothèque doit-être installée sur votre système. Sous Windows vous pourrez placer directement la "DLL" dans le dossier de vos exécutables.

Les autres bibliothèques requises sont :

  • LibWebp : Pour la prise en charge des image au format Webp

  • OpenAL, warp_oal et soft_openal : Pour la gestion audio (note : soft_openal est la version software : https://openal-soft.org), pour l'utilisation de cette dernière vous devrez le renommer en "OpenAL64" ou "OpenAL32"

  • Libmodplug : Prise en charge des format audio tracker (.mod, .st3, .xm,...)

  • LibVorbis, LibOgg : Prise en charge des formats audio OGG

  • Libmpg123 : Prise en charge du format audio MP3

Pour Windows, vous trouverez les "DLL" dans le dossiers Externals/Windows/x64 et Externals/Windows/x86.
Pour Linux vous devrez installer ces bibliothèques depuis votre gestionnaire de paquet

Installation

  1. Lancez Lazarus (pas trop fort quand même !)

  2. Dirigez vous vers le menu Paquet –> Ouvrir un paquet

  3. Choisissez en 1er le paquet bzscene_rt qui se trouve dans le dossier "Packages"

  4. Cliquez sur le bouton Compiler

Recommencez les étapes 2,3 et 4 pour les paquet dans cet ordre :

  • bzscene_tool_rt

  • bzscene_controls_rt

  • bzscene_image_rt

  • bzscene_engine_rt

  • bzscene_audio_rt, seulement si vous désirez utiliser les fonctions audio

Vous pouvez maintenant également, si vous le désirez installer les composants dans l'EDI.

  1. Dirigez vous vers le menu Paquet –> Ouvrir un paquet

  2. Choisissez en le paquet bzscene_control_dt (ce paquet est recommandé, car de nombreux exemples utilise TBZImageViewer) qui se trouve également dans le dossier "Packages"

  3. Cliquez sur le bouton Utiliser et choisissez Installer

Optionnellement vous pouvez installer les autres composants disponibles en suivant les mêmes étapes que ci-dessus avec les paquets suivant :

  • bzscene_tool_dt

  • bzscene_audio_dt - NON RECOMMANDE A CAUSE D'UN BUG LIE A L'EDI

Note : BUG : NE PAS INSTALLER LE PAQUET BZSCENE_AUDIO_DT malgré que celui-ci s'installe s'en encombre. En effet un bug est présent et génère une Erreur : "Out of memory" dès que vous souhaiterez poser une des composants sur votre fiche, sauf pour le "SoundManager".
Néammoins vous pourrez utilisez ces composants en les créant par code



Démarrer un projet avec BZScene

  1. Créez un nouveau projet "application"

  2. Dirigez vous vers le gestionnaire de projet

    1. Faites un clique droit sur "Paquets requis"

    2. Dans la boite de dialogue, en fonction des méthodes que vous souhaitez utiliser, choisissez les paquets : bzscene_rt, bzscene_image_rt, bzscene_tools_rt, bzscene_engine_rt, bzscene_audio_rt


    NOTE : Le choix des paquets insereront automatiquement les dépendances requises du paquet. Exemples :

    • Vous choisissez le paquet bzscene_image_rt, le paquet bzscene_rt sera automatiquement référencé.

    • Vous choisissez le paquet bzscene_engine_rt, les paquet bzscene_rt, bzscene_image_rt seront automatiquement référencés.


  3. Enregistrez votre projet

  4. Facultatif, mais conseillé. Dirigez vous dans les options du projet. Cliquez sur le bouton Importer et allez chercher le fichier "Lazarus_BZScene_Desktop_Config" qui se trouve dans le sous-dosssier LazProjectConfig de la bibliothèque.

    Ce fichier iimportera directement les options de compilation (debug et release), nécessaire pour la cross-compilation pour les environnements Windows et Linux 32 et 64bit.
    Il vous suffira juste de choisir le mode que vous déserez pour compiler votre projet.

  5. Insérer les nom d'unités que vous souhaitez utiliser dans la clause uses

  6. Commencez à coder comme vous le faite d'habitude



NOTE] : Dans un soucis de compatibilité et de "non frustration" les fonctions en assembleur on été désactivées.
Celles-ci sont relativement stables dans des environnement 64 bits et ont fait l'objet de tests unitaire poussés.
Vous pouvez les activer en éditant le fichier bzscene_options.inc qui se trouve dans le dossier "Source".
Dirigez vous à la ligne 93 et supprimer le point "." de la commande {.$DEFINE USE_ASM_SSE_3}.

Si vous utilisez l'assembleur, il vous faudra faire attention à l'alignement des données.

Pour ce faire vous devrez ajouter les ligne suivantes dans l'entète de vos unités (hors unités de l'application contenant des "Form" ou ce n'est pas obligatoire) :

{$ALIGN 16}
{$CODEALIGN CONSTMIN=16}
{$CODEALIGN LOCALMIN=16}
{$CODEALIGN VARMIN=16}

Si vous déclarer des variables vous devrez entourer celle-ci comme ceci :

  • Dans une classe : 

    Type
  MaClass = Class
  private
    {$CODEALIGN RECORDMIN=16}
    MonVecteur : TBZVector;
    MaMatrix : TBZMatrix;
    {$CODEALIGN RECORDMIN=4}
  end;

  • Comme constante : 

    Const
  {$CODEALIGN CONSTMIN=16}
  MonVecteur : TBZVector = (x:1.0; y:2.0; z:3.0; w:4.0);	
  {$CODEALIGN CONSTMIN=4}

  • Comme une variable locale ou globale : 

    Var
  {$CODEALIGN VARMIN=16}
  MonVecteur : TBZVector;
  MaMatrix : TBZMatrix;
  {$CODEALIGN VARMIN=4}


A Savoir (uniquement avec l'utilisation de l'assembleur) vous serez limiter à 4 (5 maximum) operations sur un seule ligne, suivant les cas.
Exemple : 

function compute : TBZVector;
Var
  {$CODEALIGN VARMIN=16}
  v1, v2, v3, v4, v5 : TBZVector;	
  {$CODEALIGN VARMIN=4}
begin
  Result := v1 + v2 * v3 + (v4 / v5);
end;

provoquera un SIGSEGV car une des données, ne sera plus alignée dans la pile. (dépacement de capacité)
Il vous faudra alors découper votre formule en de plus petite entités.
Exemple : 

function compute : TBZVector;
Var
  {$CODEALIGN VARMIN=16}
  v1, v2, v3, v4, v5 : TBZVector;	
  {$CODEALIGN VARMIN=4}
begin
  Result := (v1 + v2) * v3 ;
  v1 := (v4 / v5);
  Result := Result + v1;
  // ou Result := Result + (v4 / v5);
end;


N'hésitez pas également a entouré les différentes partie d'un calcul par des parenthèses. Le compilateur faisant assez bien les chose, cela diminue le risque d'obtenir des données mal alignées en mémoire.
