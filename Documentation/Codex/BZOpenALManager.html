Description

Contient le "Manager de son" pour la librairie OpenAL

  Les pilotes officiels d' OpenAL peuvent être téléchargés à partir du site OpenAL
(http://www.openal.org). ou de votre carte son site Web du fabricant.
Ou pouvez télécharger la version d'OpenAL Soft (http://www.openal-soft.org/)
Cette version à l'avantage d'être open source et est disponible pour toutes les plateformes
et supporte les environnements 32bits et 64bits. C'est cette version qui est utilisée
afin de conserver la compatibilité partout.

Pourquoi avoir choisi d'utiliser la version OpenAL Soft plutot l'officiel ? Les différences entre ces 2 librairies sont :
Pour la version officielle 32bits :

  • (Support EAX 2.0,3.0,4.0,5.0)

  • (Effets supportés : Reverb, EAX Reverb, Chorus, Echo, Distorion, Frequency Shifter,

  • (Filtres supportés : Low Pass)

Pour la version officielle 64bits :

  • (Support EAX 2.0,3.0,4.0,5.0)

  • (Effets supportés : Reverb, EAX Reverb)

  • (Filtres supportés : Low Pass)

Pour la version non-officielle (Soft OpenAL) 32bits et 64bits :

  • (Support EAX 2.0)

  • (Effets supportés : Reverb, EAX Reverb, Chorus, Echo, Distorion,

  • (Filtres supportés : Low Pass, High Pass, Band Pass)

Voilà c'est juste à cause des effets non supportés par la version 64bits qui à fait basculer la balance

Actuellement OpenAL sound manager supporte :

  • (Le positionnement des sons en 3D)

  • (Les filtres :

  • (Les effets :

  • (Nombre d'effets maximum par source : 4 + 1 filtre direct)

  • (Effet Doppler)

  • (Le changement du volume principal)

  • (Le changement du volume d'une source)

  • (Le changement de pitch d'une source)

  • (Le support de sons multi-cannaux (1 = Mono, 2 = Strereo, 3 et 4 = Quadriphonie, 6 = 5.1, 7 = 6.1 et 8 = 7.1 ))

  • (Le support des environnement 3D virtuels via les effets REVERB et EAX_REVERB )

Notes :
Pour l'implementation des filtres et effets voir unité BZSound.pas

Malheureusement avec les 2 librairies (officielles et soft) ont ne peux pas utiliser
les effets d'environnements virtuel 3D, car les fonctions "eaxSet" et "eaxGet"
ne sont pas disponiblent. Il nous faut donc les émuler via les effets "Reverb" EFX ou EAX. De plus ces fonctions ne fonctionnent que si le mode
"Hardware" est activé, ce qui est impossible sous Windows (du à sont fonctionnement :( )
Je n'ai pas testé sous d'autres OS comme linux, donc à voir.
Cependant sous Windows (non testé avec d'autres OS) les fonctions "EAXSet" et "EAXget"
sont disponiblent uniquement avec la version officielle d'OpenAL, mais il faut utiliser
le fichier "wrap_oal.dll" à la place de "OpenAL32.dll" mais idem cela ne change rien du fait
que le mode Hardware n'est pas disponible.

La solution est donc d'utiliser les effets REVERB et EAX_REVERB pour simuler les environnements.

Pour infos l'installateur de la version officiel d'OpenAL pour Windows sur des sytemes
64 bits place les Librairie DLL 32 bits dans le dossier c:\Windows\sysWOW64 et les librairies 64 bits dans le dossier c:\Windows\system32.
Attention : leurs noms ne changent pas et reste "OpenAL32.dll" et "wrap_oal.dll"
