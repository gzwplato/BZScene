Unité

BZCadencer

Déclaration

type TBZCadencer = class(TComponent)

Description

Composant permettant une progression automatique d'une animation.

   Envoie les événements de progression en temps réel (le temps sera mesuré en secondes).
   Ou il gardera la CPU occupée à 100% si possible (c.-à-d. si les choses changent dans votre scène).
   Le temps de progression (celui que vous verrez dans vos événements de progression)    est calculé en utilisant (CurrentTime-OriginTime) * TimeMultiplier.
   "CurrentTime" est mis manuellement ou automatiquement à jour en utilisant    "TimeReference" (le paramètre "CurrentTime" NE déclenche PAS la progression).

Description

Méthodes

Constructor Create(AOwner: TComponent); Override;
 
Destructor Destroy; Override;
 
Procedure Subscribe(aComponent: TBZCadenceAbleComponent);
 
Procedure UnSubscribe(aComponent: TBZCadenceAbleComponent);
 
Procedure Progress;

Permet de déclencher manuellement une progression.


      Le temps est traité automatiquement. Si cadencer est désactivé, cette fonction ne fait rien.

Function GetCurrenttime: Double;

Ajuste "CurrentTime" si nécessaire, puis renvoie sa valeur.

Function IsBusy: Boolean;

Renvoie True si une Progression est en cours.


      Soyez conscient que, tant que IsBusy est Vrai, le Cadencer peut envoyer de messages et appeller d'autres progressions vers des autres scènes et composants cadencés. 

Procedure Reset;

Remise à zero des paramètres et du temps.

Propriétés

property OriginTime: Double read FOriginTime write FOriginTime;

Valeur soustraite au temps actuel pour obtenir le temps de progression.

property CurrentTime: Double read FCurrentTime write SetCurrentTime;

Temps actuel (réglé manuellement ou automatiquement, voir TimeReference).

property Enabled: Boolean read FEnabled write SetEnabled Default True;

Activer / Désactiver la Candence.

      La désactivation ne provoquera pas de saut lors du redémarrage, mais fonctionne comme       une lecture / pause (c.-à-d. peut modifier OriginTime pour garder une progression douce).

property TimeReference: TBZCadencerTimeReference read FTimeReference write SetTimeReference Default cmPerformanceCounter;

Définit comment CurrentTime est mis à jour. Voir TBZCadencerTimeReference.

     Le changement dynamique de TimeReference peut provoquer un 'saut'.

property TimeMultiplier: Double read FTimeMultiplier write SetTimeMultiplier Stored StoreTimeMultiplier;

Multiplicateur appliqué à la référence de temps.

      Zéro n'est pas une valeur autorisée, et sachez que si des valeurs négatives       sont acceptés, elles peuvent ne pas être pris en charge par d'autres objets.
      La modification du TimeMultiplier modifiera OriginTime.

property MaxDeltaTime: Double read FMaxDeltaTime write FMaxDeltaTime;

Valeur maximale pour deltaTime dans les événements de progression.

      Si null ou négatif, aucun deltaTime max est défini. Sinon, chaque fois qu'un événement dont le deltaTime réel est supérieur à MaxDeltaTime, le deltaTime sera à son maximum, et le temps supplémentaire est mise en cache       par le cadencer (il n'est pas encore pris en compte dans CurrentTime).
      Cette option permet de limiter le taux de progression dans les simulations où Des valeurs élevées entraîneraient des erreurs / comportement aléatoire.

property MinDeltaTime: Double read FMinDeltaTime write FMinDeltaTime;

Valeur minimale pour deltaTime dans les événements de progression.

      Si supérieur à zéro, cette valeur spécifie le "pas" du temps minimum entre deux événements de progression.
      Cette option permet de limiter le taux de progression dans les simulations où les valeurs faibles entraîneraient des erreurs / comportement aléatoire.

property FixedDeltaTime: Double read FFixedDeltaTime write FFixedDeltaTime;

Valeur temporelle fixe pour les événements de progression.

      Si supérieur à zéro, des étapes de progression se produiront avec celles fixées au temps       delta. La progression reste basée sur le temps, donc zéro vers N événements       peut être déclenché en fonction du deltaTime réel (si deltaTime est       inférieur à FixedDeltaTime, aucun événement ne sera déclenché s'il est supérieur       à deux fois FixedDeltaTime, deux événements seront déclenchés, etc.).
      Cette option permet d'utiliser des étapes de temps fixes dans les simulations (pendant       l'animation et le rendu peuvent se produire à un niveau de fractionnement inférieur ou supérieur).

property Mode: TBZCadencerMode read FMode write SetMode Default cmASAP;

Ajuste comment les évènement de progression doivent être déclenchés. Voir TBZCadencerMode.

property SleepLength: Integer read FSleepLength write FSleepLength Default -1;

Permet de laisser du "temps" à d'autres threads / processus.

      Si SleepLength> = 0 alors AVANT chaque progression une pause se produit (voir       la procédure "Sleep" dans FPC pour plus de détails).

property OnProgress: TBZCadencerProgressEvent read FOnProgress write FOnProgress;

Evenement déclenché apres le progression.

property OnTotalProgress: TBZCadencerProgressEvent read FOnTotalProgress write FOnTotalProgress;

Evenement déclenché quand toutes les iterations avec DeltaTime fixe sont finis.
