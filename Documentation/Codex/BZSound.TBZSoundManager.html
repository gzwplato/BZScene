Déclaration

type TBZSoundManager = class(TBZCadenceAbleComponent)

Description

Classe de base pour les composants du gestionnaire de son.
Le composant sound manager est l'interface d'une API audio de bas niveau (comme DirectSound, Bass, FMod, OpenAL...)

Il ne peut y avoir qu'un seul gestionnaire actif à tout moment (Cette classe prend soin de cela).
La sous-classe doit remplacer les méthodes protégées DoActivate et DoDeActivate pour 'initialiser / unitialiser' leur couche sonore.

procedure UpdateSources; virtual;

Manual request to update all sources to reflect changes. Default implementation invokes UpdateSource for all known sources.

procedure StopAllSources;

Stop and free all sources.

procedure DoProgress(const progressTime: TBZProgressTimes); override;

Progress notification for time synchronization. This method will call UpdateSources depending on the last time it was performed and the value of the UpdateFrequency property.

function CPUUsagePercent: Single; virtual;

Sound manager API reported CPU Usage. Returns -1 when unsupported.

True if EAX is supported.

Propriétés

property Active: Boolean read FActive write SetActive default False;

Activation/deactivation of the low-level sound API

property MaxChannels: Integer read FMaxChannels write SetMaxChannels default 8;

Maximum number of sound output channels. While some drivers will just ignore this value, others cannot dynamically adjust the maximum number of channels (you need to de-activate and re-activate the manager for this property to be taken into account).

property OutputFrequency: Integer read FOutputFrequency write SetOutputFrequency default 44100;

Sound output mixing frequency. Commonly used values ar 11025, 22050 and 44100. Note that most driver cannot dynamically adjust the output frequency (you need to de-ativate and re-activate the manager for this property to be taken into account).

property Mute: Boolean read FMute write SetMute default False;

Request to mute all sounds. All sound requests should be handled as if sound is unmuted though, however drivers should try to take a CPU advantage of mute over MasterVolume=0

property Pause: Boolean read FPause write SetPause default False;

Request to pause all sound, sound output should be muted too. When unpausing, all sound should resume at the point they were paused.

property MasterVolume: Single read FMasterVolume write SetMasterVolume;

Master Volume adjustement in the [0.0; 1.0] range. Driver should take care of properly clamping the master volume.

property Sources: TBZSoundSources read FSources write SetSources;

Currently active and playing sound sources.

property UpdateFrequency: Single read FUpdateFrequency write SetUpdateFrequency stored StoreUpdateFrequency;

Update frequency for time-based control (DoProgress). Default value is 10 Hz (frequency is clamped in the 1Hz-60Hz range).

property Cadencer: TBZCadencer read FCadencer write SetCadencer;

Cadencer for time-based control.

property DistanceFactor: Single read FDistanceFactor write SetDistanceFactor stored StoreDistanceFactor;

Engine relative distance factor, compared to 1.0 meters. Equates to 'how many units per meter' your engine has.

property RollOffFactor: Single read FRollOffFactor write SetRollOffFactor stored StoreRollOffFactor;

Sets the global attenuation rolloff factor. Normally volume for a sample will scale at 1 / distance. This gives a logarithmic attenuation of volume as the source gets further away (or closer). Setting this value makes the sound drop off faster or slower. The higher the value, the faster volume will fall off.

property DopplerFactor: Single read FDopplerFactor write SetDopplerFactor stored False;

Engine relative Doppler factor, compared to 1.0 meters. Equates to 'how many units per meter' your engine has.

property Environment: TBZSoundReverbEnvironment read FSoundEnvironment write SetSoundEnvironment default seGeneric;

Sound environment (requires EAX compatible soundboard).

property UseEnvironment : Boolean read FUseEnvironment write SetUseEnvironment;
